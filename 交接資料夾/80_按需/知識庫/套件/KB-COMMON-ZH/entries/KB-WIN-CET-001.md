# KB-WIN-CET-001：Windows命令環境啟動時回報CET支援錯誤

- 修訂：`1`
- 狀態：`替代方式已驗證一次；永久修復未驗證`
- 適用：`Windows原生Codex命令環境；其他AI、WSL或雲端環境不自動套用`
- 最後觀察：`2026-07-20`

## 匹配症狀

只有出現相同或高度相近的啟動錯誤才使用：

```text
Fatal error.
Your Windows doesn't fully support CET. Please install all available Windows updates.
```

- 已觀察結束碼：`-2146233082`
- 錯誤發生在PowerShell命令真正執行前，不是命令參數錯誤。
- 未確認失敗的是`powershell.exe`或`pwsh.exe`，不得自行補成事實。

## 最低必要處理

1. 第一次命中後停止重試相同的預設命令環境。
2. 若只需讀寫或搜尋本機檔案，改用目前環境已提供且能正常使用的非PowerShell工具。
3. 已觀察案例中，持續式Node.js可用：`node:fs/promises`可讀寫檔案，`sharp`可處理PNG，既有圖片檢視工具可核對成果。
4. 只驗證本輪需要的操作；替代方式成功後繼續，不調查無關系統元件。

若工作必須使用替代工具無法完成的PowerShell、外部程式、安裝或系統命令，回報原始錯誤與受阻步驟並詢問使用者。錯誤文字建議安裝Windows更新，但這仍是未在本案例驗證的永久修復方向。

## 不得自行執行

- 不修改PowerShell執行政策。
- 不停用或變更CET及Windows安全設定。
- 不因推測調整Windows Sandbox或系統政策。
- 不反覆執行已確認會在啟動階段失敗的相同命令環境。
- 不宣稱Windows更新必定修復，除非後續實測成功。

## 成功判定

- 不再重試失敗的命令環境。
- 替代方式完成當前必要操作，並依原工作規則通過最低充分驗證。
- 沒有為此修改系統安全設定。

## 尚待確認

- 實際失敗的PowerShell可執行檔及版本。
- Windows版本、更新狀態與CET支援細節。
- 完成Windows更新並重新啟動後是否恢復。
- 相同錯誤是否影響其他工作區、帳號或電腦。
