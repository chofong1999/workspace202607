# 使用者：制度與使用說明

## 這套系統做什麼

這套系統讓AI記住一份圖片工作的長期風格、樣板、候選方案、製作流程與品質標準，避免溝通多輪才完成一張圖，下一張卻又大幅偏離。

它不是AI交接系統，也不管理員工身分。未來可作為AI工作記憶與交接平台的作圖外掛；目前版本可獨立測試，也可與同一專案中的交接資料夾協作。

## 最簡單的使用方式

第一次：

> 找到作圖系統資料夾，執行！首次建立。先說明你理解的工作、要建立的規格與缺少資料，等我確認後再寫入。

開始一張圖：

> 執行！開始作圖。先建立本張契約，再依規格製作。

測試兩種畫法：

> 執行！測試候選。把兩種方法分開記錄與驗收，不要自行選定正式方案。

開始批量：

> 執行！批次製作。先確認樣板與批次條件成立。

## 使用者通常只需決定

- 圖片目的與不能改變的內容。
- 哪些樣板或元件已經合格。
- 候選方案採用哪一個。
- 是否允許改變已確認風格。
- 是否接受AI無法客觀判定的主觀結果。

AI應自行完成可客觀檢查的部分，不把所有驗證成本轉交使用者。

## 狀態差異

- `內部QA通過待確認`：AI檢查未發現問題，但你尚未確認。
- `使用者已確認`：你已明確接受指定範圍。
- `正式完成`：內部檢查、必要的使用者確認、檔案分流及紀錄都完成。

## 規則變更

AI可以提出改善建議，但不能擅自：

- 改變已確認樣板。
- 放寬驗收條件。
- 以自己的設計偏好取代你的要求。
- 修改規則讓錯誤成果變成合格。

如果只是單張特例，應保存在`12_當前圖片與批次.md`，不自動變成長期規則。

尚未決定的多種畫法會登錄在`21_候選方案登錄.md`；它們在你確認前都不是正式標準。

`！更新規格`只改這份工作的視覺要求；`！更新作圖系統`才更新通用系統版本。兩者不能混用。

已有SOP、品牌指南或規格書時，不需要重寫一套。使用：

> 執行！同步外部規格。先登錄來源、版本、權威程度與適用範圍，再整理必要的執行摘要。

系統預設保留原文件，只引用其位置；不因導入而搬移、刪除或封存。

## 成本控制

- 普通圖片只做最低充分檢查。
- 只修改局部時，重點檢查變動區及相鄰區域。
- 新樣板、新方法、異常或最終驗收才做完整檢查。
- 每張圖只讀本張短契約及相關規則，不重讀全部歷史。
- 達到修正上限時停止並說明，不無限消耗時間。

## 與交接系統一起使用

同一工作可以把交接資料夾與作圖工作資料並列。你正常要求作圖、修改規格、管理候選／樣板、建立圖片契約、記錄QA或修改系統狀態時，不需要先另外輸入接手指令；系統會在第一次寫入前自動依現有交接制度檢查並取得工作權。

- 只讀查看、分析、規劃及檢查狀態不會接手。
- 已經是目前工作者時直接繼續，不重複接手。
- 找不到交接系統時，作圖系統可獨立運作。
- 有其他員工正在工作或交接狀態不允許接手時，系統會停止寫入並說明真正原因，不會繞過制度。
- 首次建立先做只讀分析；你確認正式建立後，第一次寫入前才自動接手。

作圖系統的重要進度必須讓交接記憶知道，但不把整份做圖規則重複貼進交接文件。使用：

> 執行！同步交接摘要。

應同步的只有：作圖系統位置與版本、視覺規格版本、外部規格版本、樣板版本、當前任務、重要決定或阻塞及下一步。詳細規則與逐張QA仍留在作圖系統。

「不得修改交接系統」只表示不得擅改交接制度核心；不代表完全不能依制度取得工作權或更新交接記憶。完成自動或手動接手的AI仍應依交接制度更新該工作的必要摘要，並在正式交接時追加日誌。

## 功能與指令

| 功能 | 明確指令 | 用途 |
|---|---|---|
| 首次建立 | `！首次建立` | 建立空白工作，或從既有SOP與成果導入 |
| 開始作圖 | `！開始作圖` | 建立本張契約後開始製作 |
| 測試候選 | `！測試候選` | 分開測試尚未定案的畫法 |
| 確認樣板 | `！確認樣板` | 登錄使用者確認的樣板及確認範圍 |
| 批次製作 | `！批次製作` | 通過樣板與批次閘門後批量執行 |
| 檢查圖片 | `！檢查圖片` | 依風險執行最低充分或完整QA |
| 更新工作規格 | `！更新規格` | 修改這份工作的視覺規格並評估影響 |
| 更新通用系統 | `！更新作圖系統` | 更新本套通用制度，不等於改工作風格 |
| 同步外部規格 | `！同步外部規格` | 核對新版SOP、品牌指南或需求書 |
| 同步交接記憶 | `！同步交接摘要` | 把重要進度與下一步同步到交接系統 |
| 查看狀態 | `！檢查狀態` | 只讀目前狀態，不修改成果 |

沒有指令符號時，AI仍可依自然語言判斷；有指令符號時，必須依既有功能及條件執行，條件不符不能擅自改做其他寫入功能。

## 0.3版的主要差異

- 寫入型功能自動依既有交接制度取得工作權；只讀不接手、已接手不重複。
- 使用者提出清楚要求後，由AI自行完成路由與最短充分契約，不要求逐步下內部指令。
- 候選比較固定共同來源與不變項，共通QA與互斥QA分離。
- 確認候選差異不再被誤解為確認整張圖或整個樣板。
- 補強系統更新的舊制度殘留排除與文件更新時間一致性。
- 維持外部規格引用優先、交接摘要最小化及可獨立運作。
