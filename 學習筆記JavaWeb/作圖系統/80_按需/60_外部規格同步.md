# 按需：外部規格同步

只在外部SOP、品牌指南、需求書、樣板集合或其他已登錄來源新增、替換或版本改變時執行。

## 1. 確認來源

1. 在`13_外部規格與來源.md`找到來源ID。
2. 核對實際位置、來源擁有者、權威程度、版本及適用範圍。
3. 來源不可讀、身分不明或疑似拿錯專案時停止，不以舊記憶補寫。
4. 只讀新舊差異所需部分；沒有可靠舊基準時才讀完整現行規格。

## 2. 分類變更

- 不改意義的文字或格式修正。
- 新增或修改工作專用規則。
- 新增候選方法。
- 樣板或元件變更。
- 驗收方式變更。
- 已淘汰或失效規則。
- 與現行規則衝突。

## 3. 同步

1. 更新`13`的版本、核對時間及同步狀態。
2. 只把日常執行必須快速取得的摘要同步到`11`，並保留來源ID。
3. 候選寫入`21`，不得混入已確認規則。
4. 樣板變更依`10_候選方案與樣板確認.md`處理。
5. 依`30_規格更新與影響評估.md`判斷既有圖片與批次。
6. 在`30_製作與QA紀錄.md`留下短紀錄；不複製整份外部文件。
7. 若有交接系統且變更影響目前工作、下一步或阻塞，依`70_交接系統協作.md`同步摘要。

## 4. 完成條件

- `13`與實際來源版本一致。
- `11`沒有與權威來源相反的摘要。
- 候選、正式規則及歷史資料仍分開。
- 受影響的圖片、樣板與批次已列出。
- 未經使用者同意，沒有移動、覆蓋或刪除外部資料。
