# 按需：批次製作

## 1. 開始條件

全部成立才能開始：

- `00`顯示允許批次。
- `11`有已確認現行規格。
- `20`有覆蓋本批次主要版型的確認樣板。
- `12`列明批次範圍、來源、成果位置及規則組。
- 沒有未解決的系統性問題。
- 來源與輸出位置不會互相覆蓋。

## 2. 批次前

1. 將圖片按版型、風險或內容差異分組。
2. 每組指定適用樣板與規則組。
3. 先處理代表性或高風險圖片。
4. 設定檢查節點；依批次風險決定，不固定每幾張。
5. 預估時間與成本；超過使用者預期時先回報。

## 3. 每張圖片

1. 建立或套用本張契約。
2. 產生非破壞性候選。
3. 執行`QA-BASIC`及必要的`QA-CHANGE`。
4. 記錄規格、樣板、方案與成果狀態。
5. 不符合時不流入正式成果。

## 4. 一致性檢查

檢查：

- 固定元件、色彩、字體、尺寸、間距及輸出形式。
- 內容分類及說明語氣。
- 同類情況是否採用相同規則。
- 是否出現後期成果逐漸偏離樣板。

一致性檢查不能只確認「每張都不一樣地通過」；要實際與確認樣板及同批成果比較。

## 5. 停止條件

- 相同問題第二次出現。
- 新內容超出確認樣板。
- 規格或樣板版本在批次中改變。
- 來源、成果位置或檔名有混淆風險。
- 內部QA與實際觀看結果矛盾。

停止時保留已完成且可證明合格的成果，不要求無關圖片重新檢查。

## 6. 批次完成

- 統計正式完成、等待確認、未通過及暫停數量。
- 執行約定的批次一致性檢查。
- 確認使用者入口只有應交付成果。
- 更新`12`與`30`。
- 需要最終整批驗收時再執行`QA-FULL`，不把一般接手檢查擴大成全面重驗。
