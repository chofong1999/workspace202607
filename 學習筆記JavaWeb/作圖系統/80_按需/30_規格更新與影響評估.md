# 按需：規格更新與影響評估

## 1. 先判斷變更類型

- `文字釐清`：不改變實際成果或驗收。
- `工作狀態更新`：只更新進度、路徑或當前任務。
- `視覺規格變更`：改變長期畫法、固定元件、內容或驗收。
- `樣板變更`：改變使用者確認的視覺基準。
- `系統流程變更`：改變本通用系統；工作中的AI不得自行執行。

只有`視覺規格變更`增加視覺規格版本；只有確認樣板變更增加樣板版本。單張成果修正只增加成果修訂。

## 2. 權限

AI可以直接：

- 修正不改義的錯字、斷鏈或明確引用錯誤。
- 更新使用者已明確決定的規則。
- 更新當前狀態與成果紀錄。

AI必須先詢問：

- 新增、放寬、取代或取消已確認規則。
- 改變確認樣板。
- 將單次特例變成長期規則。
- 變更可能使既有正式成果不再合格。

## 3. 影響評估

每次視覺規格或樣板變更，列出：

1. 問題與變更來源。
2. 被取代規則。
3. 新規則及生效範圍。
4. 哪些樣板受影響。
5. 哪些成果受影響。
6. 舊成果處理：
   - 不受影響。
   - 只更新標記。
   - 需要`QA-CHANGE`。
   - 需要`QA-FULL`。
   - 需要重做。
7. 批次能否繼續。

不得因資料夾或紀錄格式改變，自動要求重做視覺內容。

## 4. 一致性更新

完成變更前檢查：

- `00`狀態、規格版本與樣板版本。
- `11`現行規則。
- `12`當前契約、方案及下一步。
- `20`樣板與元件。
- `21`候選方案及其採用／淘汰狀態。
- `30`變更與QA紀錄。
- 檔名、成果標記及使用者入口。

修改A影響B時，必須在同一次變更中更新B；無法完成時保持`暫停待處理`，不得留下看似可用的半套狀態。
