# 通用作圖流程

- 系統版本：`0.3`
- 適用範圍：需要穩定延續風格、批量製作、反覆修正或由AI持續製作的圖片工作

## 1. 核心原則

1. 先確定目的與來源，再決定畫法。
2. 原始素材、候選成果、正式成果與內部QA分開保存。
3. 已確認樣板是視覺基準；AI不能因個人判斷自行換風格。
4. 每張圖片只載入本張適用規則，不必重讀全部歷史。
5. 未確認的畫法以候選方案管理，不冒充正式標準。
6. 技術QA通過不等於使用者已確認。
7. 相同錯誤重複出現時，先處理系統性原因，不繼續擴大批次。
8. 檢查強度依風險與變動範圍決定，不固定對所有圖片做最重檢查。
9. 使用者提出清楚的作圖要求後，AI自行完成必要路由、工作權檢查與短契約，不要求使用者逐一啟動內部步驟。

## 2. 標準生命週期

### A. 建立工作

1. 說明圖片用途、受眾、輸入、輸出及不可改變內容。
2. 建立來源與成果位置。
3. 若已有外部SOP或規格，先登錄至`13_外部規格與來源.md`，再將必要的執行摘要寫入`11_本工作視覺規格.md`。
4. 若已有合格圖片，判斷哪些部分可登錄為確認樣板或元件。
5. 沒有確認樣板時，進入候選測試；不得直接批次。

### B. 建立本張契約

每張圖片或同質小批次開始前，在`12_當前圖片與批次.md`填寫：

- 任務代號、來源、預定成果。
- 適用的規格與樣板版本。
- 3～12項本張強制規則；超過時改引用已確認規則組。
- 可調整範圍與禁止改變內容。
- 候選方案或正式方案。
- QA層級與完成證據。

這份短契約是工作中的主要執行清單，不能用來創造未經確認的新規則。

契約只寫本張真正需要的內容。規格摘要已同步且來源版本未變時，不重讀完整外部規格；引用規則組即可。

### C. 製作

1. 複製來源或建立非破壞性工作版本。
2. 依本工作規格規定的元件順序製作。
3. 遇到規則未涵蓋情況，先判斷是否能在可調整範圍內處理。
4. 若會改變已確認風格、內容、驗收條件或資料位置，停止相關部分並詢問。
5. 不得因為「這樣較合理、較完整、較好看」而跳過已確認規則。

### D. 檢查

1. 每張圖片至少做`QA-BASIC`。
2. 只修改局部時，再做`QA-CHANGE`檢查變動區與受影響邊界。
3. 新樣板、新候選、新規則、系統性異常或使用者要求時做`QA-FULL`。
4. 關鍵驗證不得完全重用產生時的同一假設或推算結果。
5. 將結果寫入`30_製作與QA紀錄.md`。

### E. 使用者確認

需要主觀風格或規格決策時：

1. 只提供足以比較的候選方案。
2. 明確列出差異與影響，不替使用者選擇。
3. 使用者確認時記錄確認的是整張、某個元件、某項規則或某種版型。
4. 未被確認的部分不得由AI自行擴張成標準。
5. 確認某個候選只代表確認記錄中明列的差異範圍；不自動確認整張圖片、其他元件、版型或整批成果。

### F. 批次

1. 只有確認樣板覆蓋該版型時才允許批次。
2. 每張仍做基本檢查；一致性抽查不能取代單張基本檢查。
3. 新情況超出樣板適用範圍時，暫停該類圖片並建立候選。
4. 重複錯誤依`80_按需/20_異常與失敗處理.md`停止或降級。

### G. 完成

完成時必須同時符合：

- 本張強制規則均通過或有使用者明確接受的例外。
- 沒有把未知或未驗證內容宣稱為通過。
- 成果狀態、位置、規格版本、樣板版本及QA紀錄一致。
- 使用者成果入口不混入內部QA、失敗候選或歷史資料。

若同一專案使用交接系統，完成重要里程碑或準備離開時，依`80_按需/70_交接系統協作.md`同步最小摘要；不把整份作圖規則複製進交接資料。

## 3. 有限迭代

- 預設單張自行修正上限為3輪；`11`可另行設定。
- 每輪必須記錄新發現、修正目標與結果，不能無差別重做。
- 相同原因連續失敗兩次時，不繼續同策略；改讀異常流程。
- 達上限後標記`達迭代上限`，回報阻塞及可選下一步，不得把上限當成通過。
- 使用者要求再試一次時，可以增加一輪，但仍不得重複完全相同且已證明失敗的方法。

## 4. 候選比較的最低契約

比較兩個以上方案時，先建立一份共用短契約：

- 使用同一來源或明確等價的輸入。
- 列出各候選唯一允許不同的項目。
- 其他版型、內容、元件與輸出條件保持一致。
- 共通規則只驗證一次；互斥規則分別驗證，不混用QA。
- 成果清楚標示候選ID，與正式成果分流。
- 使用者選定候選後，只把已確認範圍同步為正式規則。

缺少可公平比較的共同來源或差異邊界時才詢問；不要求使用者另外輸入候選測試指令。

## 5. 不適合啟用完整系統的情況

一次性、低風險、沒有延續需求的簡單圖片，可只使用通用品質規則與單次本張契約，不必建立樣板、批次與長期紀錄。是否需要完整建立，以避免返工所能節省的成本判斷。
