# 批次生產：進入閘門與回歸 QA

- 適用工作：`WRK-20260720-408T2R4T`
- 方法版本：`1`
- 目的：把已發生的錯誤轉成每批必跑的檢查，避免只修當次圖片後在下一專案重犯。

## 一、批次進入條件

以下六類全部 PASS，且使用者已確認目前試作，才允許在 `12_當前圖片與批次.md` 將批次改為允許：

### G1 來源與分頁

- 來源檔、專案、Eclipse 標籤及原圖一一對應。
- 記錄來源總行數與每頁首末行。
- 相鄰頁缺行數 0、重複頁數 0。
- 第二頁起每頁至少新增一行來源內容。
- 只差游標、選取列、捲軸或狀態列的相同內容不得視為新頁。

### G2 語法分類

- 依來源碼列出每頁實際存在的 HTML、CSS、JSP 指令／註解、JSTL、JSP Action、Java scriptlet、EL、JavaScript。
- 框線、括號、說明與固定八項圖例的分類一致。
- JavaScript 內文與 HTML attribute 外層不得混成同一分類框。

### G3 標記幾何

- 框線穿字、框線共線、不同色框線交叉及中心距不足項目均為 0。
- 跨畫面長引線與不同色連線交叉均為 0。
- 編號互疊、編號碰標記、同類重複編號及不完整矩形均為 0。
- 括號與其他不同色標記距離不足 4 px 的項目為 0。
- 同一分類同一頁預設只保留一個代表編號。

### G4 EL 獨立證據

- 來源碼 EL 數量與原圖 OCR 數量一致。
- 每個 EL 都保存 `$`、`{`、`}`、左右鄰字及像素框。
- `$`與`}`前景像素完整改色，左右鄰字及 EL 遮罩外變動為 0。
- 製圖與 QA 不得共用固定字寬或同一組推算座標。

### G5 成果與分流

- 每張成果可開啟、尺寸正確且目標檔案標籤可辨識。
- 使用者入口只保留目前有效圖片，不混入 JSON、QA、失敗版、重複頁或歷史版。
- 失敗成果及撤回原因保留在內部 QA；不得仍標記 PASS。
- Java Web 原始碼與無標註原圖雜湊保持不變。

### G6 回歸案例

每次批次前重跑下列已知錯誤類型：

| 回歸 ID | 已知錯誤 | 必須得到的結果 |
|---|---|---|
| REG-01 | EL `$`或`}`漏染、引號／標籤誤染 | 目標完整；鄰字與遮罩外變動 0 |
| REG-02 | 背景或整個字元格被染色 | 背景變動 0 |
| REG-03 | 拉到底頁面的垂直偏差 | 所有框、括號與 EL 共用該頁實測基準 |
| REG-04 | 長引線交錯 | 長引線 0、不同色交叉 0 |
| REG-05 | 不同色框線重疊／貼合 | 重疊、交叉、中心距不足均 0 |
| REG-06 | 編號互疊或同類編號過多 | 元件碰撞 0；同類每頁預設 1 個 |
| REG-07 | 空間不足時形成斷線矩形 | 不完整矩形 0；改用明確括號 |
| REG-08 | EL、JavaScript 等括號過近 | 不同色標記距離不足 4 px 為 0 |
| REG-09 | 長檔相鄰頁內容重複 | 每一後續頁新增來源行；重複頁 0 |
| REG-10 | 檔名、來源、標籤或頁序錯配 | 四者一一對應 |

## 二、失敗處理

1. 任一閘門失敗，立即停止該批；尚未檢查的頁面不得繼續產出。
2. 把受影響成果標記為 FAIL 或 WITHDRAWN，不能保留 PASS。
3. 記錄原始症狀、根因、影響範圍及為何舊 QA 沒抓到。
4. 把新錯誤轉成可重跑的檢查，加入本文件的回歸表或既有規則。
5. 永遠從原始截圖或已驗證的 EL 覆色基底重做，不在錯誤成品上補畫。
6. 重跑新檢查及 REG-01～REG-10 全部案例。
7. 全部通過並同步 IMGLOG、當前狀態及交接摘要後，才可重新請使用者檢視或恢復批次。

## 三、批次 QA 最低紀錄

- 工作與圖片任務代號。
- 視覺規格版本。
- 來源檔數、有效頁數、排除頁數及原因。
- G1～G6 每項 PASS／FAIL 與證據位置。
- REG-01～REG-10 的執行結果。
- 新發現錯誤、規則更新及重新執行結果。
- 使用者試作確認狀態。
- 批次是否允許；若否，唯一真正阻塞原因。
