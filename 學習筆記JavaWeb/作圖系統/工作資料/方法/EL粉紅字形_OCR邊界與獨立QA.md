# EL 粉紅字形：OCR 邊界與獨立 QA 方法

- 適用工作：`WRK-20260720-408T2R4T`
- 適用候選：`CAND-EL-001／EL 粉紅字形`
- 方法版本：`1`
- 建立依據：`SRC-SPEC-001／SOP 2.4`
- 成功實例：`IMG-MVPRODUCT0721-COLOR-001`

> 本文件保存可重複的實作方法，不代表 CAND-EL-001 已成為跨專案正式方案。

## 為什麼需要這個方法

固定字寬、字元索引乘格寬及使用同一組推算座標自我驗證，曾造成 `$`漏染、`}`錯位、外側引號誤染及假 PASS。製圖座標與驗證證據必須獨立。

## 一、來源分析

1. 讀取真正的 JSP 原始碼，逐行找出所有 `${...}`。
2. 記錄每個運算式的來源行號、完整文字、同一行的出現順序，以及左右相鄰來源字元。
3. 每張原圖分別量測可見行號與字形上下界。一般頁或拉到底頁都必須以該張原圖重新取得垂直錨點；不得只沿用其他圖片的固定偏差。

## 二、從原圖取得獨立字元框

1. 對每個含 EL 的實際畫面行做單行 OCR，不對成品或既有製圖座標做 OCR。
2. 已驗證可用的處理方式：
   - 使用本機 `tesseract.js`。
   - 單行辨識模式。
   - 原圖行裁切後放大 4 倍。
   - 使用 nearest-neighbor 放大，再做灰階與正規化。
   - 白名單保留 `$`、`{`、`}`、引號、運算子、英數字及 JSP 常用標點；排除 `%`，避免 `$`被誤認為 `%`。
   - 要求輸出逐字元 bounding box，不只取得整行文字。
3. 只接受「`$`後緊接 `{`，再找到對應 `}`」的字元序列。
4. 同一行 OCR 得到的 EL 數量必須與來源碼數量完全相同；依出現順序一一配對。
5. 每個 EL 至少保存 `$`、`{`、`}`字元框、完整左右界、來源左右鄰字及 OCR 字元序列。
6. OCR 缺少 `$`、`{`、`}`、數量不符、順序不明或邊界互相重疊時，該頁立即停止，不得改用固定字寬補猜。

## 三、粉紅覆色

1. 每個 EL 只使用該張原圖 OCR 得到的 `$`左界至 `}`右界。
2. 垂直範圍使用該行原圖實際字形上下界。
3. 在 EL 矩形內逐像素比較該行局部背景：
   - 與局部背景相同的像素保持不變。
   - 只有原始字形與抗鋸齒像素改為粉紅色階。
   - 依原始像素與背景的對比保留亮度層次，不使用重打文字、實心底色或整塊覆蓋。
4. 製圖永遠從無標註原圖開始；修訂時不得在舊成品上層層覆蓋。

## 四、獨立 QA

每個 EL 必須分開驗證，不得合併成整頁總數後通過。

1. `$`原始前景像素數 = `$`實際改色像素數。
2. `}`原始前景像素數 = `}`實際改色像素數。
3. 完整 `${...}`內所有目標前景像素均已改色。
4. EL 左側鄰字區變動像素 = 0。
5. EL 右側鄰字區變動像素 = 0。
6. 所有 EL 矩形聯集以外的覆色階段變動像素 = 0。
7. 加入分類框、標記及說明後，原始程式字形除 EL 外的變動像素及分類框穿字像素均須為 0。
8. 100% 檢查全圖；200% 檢查框線與配置；400% 逐個檢查 `$`、`{`、內容、`}`及左右鄰字。

任何一項不成立即為 FAIL；不得因大部分運算式正確而放過單一錯誤。

## 五、QA 紀錄最低欄位

- 來源圖片與來源檔案。
- 行號、來源運算式與 OCR 字元序列。
- `$`、`}`與完整運算式像素框。
- 左右來源鄰字。
- `$`與`}`前景像素數及改色像素數。
- 左右鄰字變動像素數。
- EL 聯集外變動像素數。
- 100%／200%／400% 結果。
- 候選 ID、輪次與最終狀態。

- `$`與`}`目標字形像素全部改色。
- 左右鄰字變動像素全部為 0。
- EL 覆色遮罩外變動像素為 0。
- 分類框穿字像素為 0。
